1 Headings

  • Use '*' to make a heading.
  • Use '**' to make a subheading and so on..
  • Another subheading. Automatically insert a sub-heading by hitting <M-RET> in insert mode. Hitting 'o' in normal mode automatically inserts bullet points

-* Some dummy content
-* More dummy content

  • Use gj/gk to move from heading to heading ignoring the content within
  • Use <M-j> <M-k> to move headings up or down
  • Use "S-Tab" to cycle through headings collapse/expansion
  • Use ',' which opens a list of available options for the current major mode. Pretty useful for org at least

2 Todo

2.1 Here's a todo list with bullet points:[2/3]

  • [X] Do thing
  • [X] Do another thing
  • [ ] And another
    The syntax is pretty WYSIWYG. Make sure to hit "C-c C-c" or "SPC m ," or ", ," to update calculations or checkmarks.

2.2 Here's another todo list using headings[66%]:

2.2.1 DONE Item 1

2.2.2 DONE Item 2

2.2.3 TODO Item 3

Use 'SPC T T' to cycle through "TODO/DONE" status. Use "[%]" to add a percentage completion status
Use ", s" to schedule a TODO item. Schedules can then be viewed in Agenda. Use ", d" to add a deadline. Note: these shortcuts may change based on the spacemacs version, so double-check before using them.

2.3 You can define custom status instead of the default "TODO" and "DONE"

  • Set this at the beginning of a line #+SEQTODO: TODO(t@) NEXT(n) DOING(o) | DONE(d!) and run ", ,"
  • (setq org-todo-keywords '((sequence "TODO(t@)" "NEXT(n)" "DOING(o)" "|" "DONE(d!)" "CANCELLED(c!)")))
    Everything after the "|" character is considered as a completed task

The letter in the bracket is the hotkey associated with the keyword
@ - log a timestamp when and a note when you enter this keyword
! - log a timestamp when you leave this keyword

2.4 You can archive completed tasks in a separate file instead of deleting them by setting #+ARCHIVE: filename.org

4 Tables

Example table:

Heading 1 Heading 2 Heading 3
Row 1 Column 1 Row 1 Col 2 Row 1 Col 3
Row 2 Col 1 Row 2 Col 2 Row 2 Col 3

To add a table simply type the column headings separated by pipe character and press "TAB". Formatting will be taken care of automatically. Use "TAB" and "S-TAB" to cycle through entries

Name Gender Address Age Company
John M SF 22 Google
Mary F NY 30 Wells Fargo
Suraj M Chicago 33 Target

Use <M-arrowkeys> to move columns left or right, or rows up or down. Tables can also be used as a spreadsheet. Check this link for more info: https://orgmode.org/worg/org-tutorials/org-spreadsheet-intro.html.

5 Source blocks

(setq toggle-truncate-line t)

Source blocks provide proper highlighting for the language, and also get formatted properly when exporting as HTML or PDF. They can also be executed from within org mode (lookup: Literate programming), but that's something to look into later.
For inline formatting, you can use tilde character for monospace formatting: include <stdio.h>

6 Agenda

Agenda feature can be used to view TODO, calendar events etc.
To add a file to be tracked by agenda, use 'C-c ['. You can also customize the org-agenda-files function
Use ", a" to open agenda view.

  • t to view only TODO items. "S-f" in agenda view to enable follow mode.
  • a to view agenda for the current week. Use 'f' and 'b' to move calendar forwards or backwards

7 Drawers
